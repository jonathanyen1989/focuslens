# Visual Elements

Elements are thinking tools, not decorations.

handdrawnyl uses a controlled set of recurring visual elements.

Element priority:

1. Thinking / Management Elements
2. System / Data Elements
3. Project / Execution Elements
4. Presentation Support Elements

Do not randomly mix all element types in one image. Choose only the elements that directly support the selected illustration type and cognitive anchor.

## Thinking / Management Elements

Use for cognitive insight illustrations, leadership principles, meeting takeaways, management thinking, question trees, decision trees, problem framing, review and reflection, hidden constraints, and leverage points.

Common elements:

- whiteboard
- question tree
- decision fork
- sticky notes
- thinking cards
- key question card
- hand-drawn arrows
- circles and highlights
- magnifier
- root cause marker
- hidden layer / iceberg
- leverage point marker
- before / after thinking panels
- simple framework boxes
- notebook margin notes

## System / Data Elements

Use for structure explanation illustrations, AI Agent workflows, Skill architecture, automation processes, Power BI / dashboard logic, data platforms, system maps, feedback loops, and input-output relationships.

Common elements:

- data flow line
- node
- agent node
- tool card
- dashboard panel
- database cylinder
- pipeline
- input module
- output module
- feedback loop
- control tower
- signal wave
- system map
- knowledge graph
- connector line
- automation loop
- validation checkpoint

## Project / Execution Elements

Use for project management visuals, KPI tracking, risk review, milestone planning, delivery management, owner-action mapping, testing workflow, review loops, and project governance.

Common elements:

- timeline
- milestone marker
- risk radar
- risk flag
- kanban board
- task card
- owner card
- checklist
- blocked sign
- delivery path
- review loop
- status board
- priority stack
- action tracker
- dependency line
- escalation marker

## Presentation Support Elements

Use for slide-ready, web-ready, or video-friendly visuals.

Common elements:

- central model
- title card
- section divider
- large idea card
- simple frame
- visual summary box
- side annotation
- spotlight area
- soft background shape
- minimal badge
- highlight ribbon
- comparison panel
- three-part structure
- before / after panel

## Element Combination Rules

- Use 3-6 core elements per image.
- Use fewer elements for cognitive insight illustrations.
- Use slightly more elements for structure explanation visuals.
- Use only elements that directly support the cognitive anchor.
- Do not combine thinking, system, and project elements unless the content clearly requires it.

## Negative Element Rules

Avoid stacking too many icons, dense PPT-style architecture diagrams, decorative icons, unrelated visual metaphors, complex room/office/city/architecture backgrounds, unnecessary detailed device mockups, overusing arrows, too many labels, turning every system into a pipeline, turning every management topic into a checklist, turning every AI topic into a robot scene, and random technology symbols.
