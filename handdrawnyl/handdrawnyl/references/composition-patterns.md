# Composition Patterns

Composition starts from the cognitive anchor, not from the drawing style.

The goal of handdrawnyl composition is to turn one cognitive anchor into one clear visual structure.

Start from:

- What is the key judgment?
- What changed in understanding?
- What structure is hidden behind the words?
- What decision, system, or relationship should become visible?

## Illustration Type Priority

1. Cognitive Insight Illustration
2. Structure Explanation Illustration
3. Presentation Visual
4. Project Management Visual

When uncertain, prefer a cognitive insight illustration over a process chart.

## Cognitive Insight Illustration Patterns

### Surface vs System

Use when visible issues are symptoms and the real cause is hidden in the system.

Composition:

- front layer: visible issues, symptoms, scattered problems
- back layer: underlying system, root cause, hidden constraints
- figure action: pointing, circling, revealing, mapping
- elements: magnifier, hidden layer, system nodes, question cards, root cause marker

### Question Tree

Use for problem framing, decomposition, or better questioning.

Composition:

- root: vague original question
- branches: sub-questions
- leaves: decision points or action directions
- figure action: writing, splitting, circling key branch
- elements: whiteboard, tree branches, sticky notes, question marks, highlight circles

### Before vs After Thinking

Use when the content shows a shift in thinking.

Composition:

- left: old thinking, surface-level view
- right: new thinking, structured view
- center: transition arrow or turning point
- figure action: moving from old board to new board, redrawing the model

### Decision Fork

Use for decision logic, trade-offs, or alternative paths.

### Hidden Constraint

Use when obvious effort does not work because a hidden constraint exists.

### Leverage Point

Use when one key action can influence the whole system.

## Structure Explanation Illustration Patterns

### Input-Process-Output

Use for flow from input to processing to output.

### Feedback Loop

Use for learning, iteration, review, or continuous improvement.

### Agent Workflow

Use for human-AI collaboration, Agent tasks, tool use, or skill orchestration.

### System Map

Use for relationships between modules, teams, systems, or concepts.

### Data Pipeline

Use for data collection, cleaning, transformation, analysis, dashboarding, or decision support.

## Presentation Visual Patterns

### Big Idea Visual

One central metaphor or model, one short title or phrase, 2-4 supporting labels, balanced white space.

### Framework Page

Central framework, 3-5 parts, short labels, light annotations.

### Visual Summary

One central theme, 3-5 surrounding insight cards, simple visual hierarchy.

## Project Management Visual Patterns

### Risk Radar

Radar circle or quadrant, risk points, severity / urgency axis, figure identifying high-risk area.

### Milestone Roadmap

Timeline, milestone markers, risk points, owner / action cards.

### Owner-Action Matrix

Owner cards, action cards, connection lines, status markers, review loop.

## Metaphor Strategy

Use clear cognitive, structural, system, and workplace metaphors. Do not use weirdness as the goal. A good metaphor should make the insight easier to understand within 3 seconds.

## Shot Selection Rules

1. Do not illustrate every paragraph.
2. Do not average visuals across content.
3. Look for cognitive anchors.
4. Prefer points with judgment, contrast, structure, decision, hidden system, or action logic.
5. Avoid weak decorative visuals.
6. Each shot must answer: why does this point deserve a visual?
7. Prefer fewer stronger shots over many weak shots.

## Layout Rules

1. One image = one main cognitive anchor.
2. Use 3-6 core visual elements.
3. Keep enough empty space.
4. Use clear hierarchy.
5. Put the main structure at the center or slightly off-center.
6. Use arrows and grouping only when they clarify logic.
7. Keep labels short.
8. Keep figure supportive, not dominant.
9. The visual should be understandable within 3 seconds.

## Anti-Pattern Rules

Avoid turning every idea into a workflow, every management topic into a checklist, every AI topic into a robot scene, every system into a dense architecture diagram, every presentation visual into a PPT infographic, using the recurring figure as a mascot, adding icons that do not support the cognitive anchor, overusing arrows/labels, mixing too many metaphors, copying previous example compositions, or keeping Xiaohei-style weirdness / black solid-body character logic.
