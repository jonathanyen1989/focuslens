# Style DNA

handdrawnyl is a hand-drawn note-style visual system for turning cognitive anchors, management insights, system logic, AI workflows, data processes, and project thinking into clean, structured, memorable explanation visuals.

It should feel like a smart person drawing on a whiteboard or notebook to explain a complex idea clearly.

It is not Xiaohei style. It is not a cute cartoon. It is not a traditional PPT infographic. It is not a commercial poster. It is not a photorealistic illustration.

## Visual Mix

- 50% hand-drawn notebook feeling
- 30% enterprise-level structure
- 20% AI / data / technology elements

## Overall Look And Feel

The image should have clean hand-drawn lines, black or dark gray primary linework, slight hand-drawn wobble, clear structure and hierarchy, enough white space, short handwritten-style labels, restrained accent colors, simple visual metaphors, light thinking-note feeling, and enterprise-readable layout.

## Color Modes

### Mode B: Tech Accent Mode

Use for AI Agent, Skill, automation workflow, Power BI, dashboard logic, data platform, system map, or structure explanation visual.

Palette:

- Background: white or very light gray
- Line: black or dark gray
- Accent: blue, cyan, purple

Rules:

- Use accents sparingly.
- Do not use large neon gradients.
- Do not create cyberpunk or sci-fi overload.

### Mode C: Thinking Note Mode

Use for cognitive insight, meeting takeaway, leadership principle, problem tree, decision tree, review diagram, or project thinking.

Palette:

- Background: white or very light warm gray
- Line: black or dark gray
- Blue: structure, path, relationship
- Yellow: highlight, sticky note, key point
- Orange: risk, warning, blocker, tension

### Mode D: Soft Color Sketch Mode

Use for presentation visual, knowledge article illustration, video page, or friendly explanation visual.

Palette:

- Background: white or soft light background
- Line: black or dark gray
- Fill: soft blue, soft green, soft purple, soft yellow

Rules:

- Keep saturation low.
- Keep the image professional.
- Do not make it childish or overly cute.

### Mode E: Custom Mode

Use when the user explicitly specifies brand colors, project colors, theme colors, special color constraints, or extreme visual scenario.

User-specified colors take priority, while preserving clean hand-drawn linework and clear visual hierarchy.

## Dynamic Line Style

### Cognitive Insight Illustration

Use whiteboard marker or notebook sketch lines. Slightly rougher lines are allowed. Use circles, arrows, underlines, quick boxes, and thinking traces.

### Structure Explanation Illustration

Use clean hand-drawn linework with slight wobble. Keep modules, arrows, and relationships readable.

### Presentation Visual

Use clean, balanced sketch lines. The image should feel polished, stable, slide-ready, and web-ready.

### Project Management Visual

Use clean sketch lines with whiteboard-style annotations. Risk markers, milestone marks, owner cards, and timeline arrows should be readable.

## Dynamic Background Rule

Default background: very light gray or very light warm white.

- Cognitive Insight Illustration: very light warm white, subtle notebook feeling, or faint dot/grid texture.
- Structure Explanation Illustration: pure white or very light gray.
- Presentation Visual: pure white, very light gray, or very light warm white.
- Project Management Visual: very light gray, very light warm white, or subtle whiteboard texture.

Avoid heavy paper texture, dark background, colorful background blocks, obvious notebook decoration, and patterns that reduce readability.

## Text Label Rule

Use minimal handwritten-style labels by default.

- Cognitive Insight Illustration: 1 main phrase + 2-4 short labels.
- Structure Explanation Illustration: 3-6 short module names or step labels.
- Presentation Visual: one short title, several key labels, concise callouts.
- Project Management Visual: slightly more labels allowed, but all text must remain short and scannable.

General:

- Prefer nouns, verbs, and short phrases.
- Avoid dense PPT-style text blocks.
- If long explanation is needed, keep it outside the image.
- Text should look like hand-drawn annotations, not formal slide typography.

## Layout Principles

1. One image should express one main cognitive anchor.
2. Use visual hierarchy: main idea first, supporting elements second.
3. Keep enough white space.
4. Use 3-6 core visual elements only.
5. Do not fill the whole canvas.
6. Prefer simple structure over decorative details.
7. Use arrows, grouping boxes, and spatial relationships to explain logic.
8. The recurring figure should support the structure, not dominate it.
9. The image should be readable within 3 seconds.

## Style Calibration

Use assets/examples only for low-frequency handdrawnyl style calibration.

Examples should calibrate line density, layout balance, color restraint, background texture, figure usage, label density, and visual clarity.

Do not copy example compositions directly. Do not use Xiaohei-style examples. Old Xiaohei example images must be removed from this skill to avoid style contamination.

## Negative Style Rules

Avoid Xiaohei style, black solid-body characters, absurd / grotesque weirdness, cute cartoon, childish illustration, emoji-like or sticker-like style, strong manga/anime/comic/storyboard style, complex PPT-style infographics, large text blocks, commercial posters, 3D rendering, glossy glassmorphism, photorealistic images, excessive neon/cyberpunk effects, complex backgrounds, and too many decorative elements.
