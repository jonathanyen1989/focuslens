# Visual Figure

The handdrawnyl figure is a stable neutral hand-drawn line figure.

The figure is a thinking operator, not a mascot.

It is not Xiaohei. It is not a cartoon character. It is not the main IP of the image.

The figure helps explain cognitive anchors, management actions, system operations, and collaboration logic.

## Base Figure Design

The base figure should be simple, neutral, hand-drawn, line-based, non-cute, non-realistic, and non-mascot-like.

Visual characteristics:

- round or slightly oval head
- minimal face or no facial details
- simple line body
- simple arms and legs
- black or dark gray thin sketch lines
- no solid black body
- no exaggerated head-body ratio
- no big eyes
- no expressive cartoon face
- no decorative costume

## Role Variations

### Whiteboard Thinker

Use for cognitive insight, leadership principle, meeting insight, question tree, decision tree, and management thinking.

Props: marker, whiteboard, sticky notes, question cards, magnifier, arrows, circles.

Actions: writing, circling key issues, splitting one problem into branches, connecting causes and effects.

### Project Operator

Use for project management, KPI review, risk tracking, milestone management, and review loop.

Props: checklist, kanban board, timeline, risk radar, owner card, milestone marker.

Actions: moving task cards, checking boxes, pointing at risk, connecting owner and action.

### AI Agent Collaborator

Use for AI Agent workflow, Skill system, automation process, Power BI / data workflow, and toolchain map.

Props: agent node, data flow, dashboard panel, tool card, automation loop, connector lines.

Actions: connecting agent nodes, reviewing dashboard signals, handing data to an agent, validating output.

### System Observer

Use for system map, feedback loop, control tower, signal map, organization operating model, and risk radar.

Props: control panel, radar, system nodes, signal waves, feedback arrows, monitor cards.

Actions: observing system signals, identifying abnormal node, adjusting feedback loop, marking leverage point.

## Figure Actions

Preferred actions: observe, write, circle, connect, split, compare, prioritize, validate, push, pull, sort, check, monitor, adjust, map, review, question, decide.

Avoid passive decorative poses.

If the figure can be removed and the image still means exactly the same thing, the figure is probably decorative and should be redesigned or removed.

## Size And Placement

- The figure should usually be small to medium-sized.
- The figure should not occupy more than 20-30% of the canvas.
- In structure explanation visuals, the figure should be smaller and supportive.
- In cognitive insight visuals, the figure can be more visible if it represents active thinking.
- Avoid placing the figure as a centered mascot.

## Negative Figure Rules

Avoid Xiaohei-style black solid character, black filled body, cute/childish/emoji/sticker/mascot-like figure, big eyes, exaggerated expressions, cartoon faces, realistic human anatomy, detailed clothing, costumes, identity markers, main IP/hero character, manga/anime/comic/chibi style, emotionally dramatic figure, or a figure that dominates the visual structure.
